# Tab Components Guide

LeadCMS provides powerful tab components that make documentation more interactive and user-friendly. The main components are `OSTabs` (with OS detection) and `CodeTabs` (for any tabbed content with grouping and persistence).

## Available Tab Components

### 1. OS-Specific Tabs (`OSTabs`)

Automatically detects the user's operating system and shows the relevant tab first. Perfect for installation instructions and OS-specific commands. This is the only specialized tab component with unique functionality.

<OSTabs>
  <OSTab os="Linux">
    ```bash
    # Install on Linux
    sudo apt update
    sudo apt install nodejs npm
    npm install -g @leadcms/cli
    ```
  </OSTab>
  <OSTab os="macOS">
    ```bash
    # Install on macOS
    brew install node
    npm install -g @leadcms/cli
    ```
  </OSTab>
  <OSTab os="Windows">
    ```powershell
    # Install on Windows
    # Download Node.js from nodejs.org
    npm install -g @leadcms/cli
    ```
  </OSTab>
</OSTabs>

**Key Features:**
- ✅ **Auto-detection** - Detects Windows, macOS, Linux, Ubuntu
- ✅ **Smart ordering** - Detected OS appears first
- ✅ **User preferences** - Remembers manual selections
- ✅ **Alias support** - Ubuntu maps to Linux automatically

**Usage:**
```mdx
<OSTabs>
  <OSTab os="Linux">
    Linux-specific content here
  </OSTab>
  <OSTab os="macOS">
    macOS-specific content here
  </OSTab>
  <OSTab os="Windows">
    Windows-specific content here
  </OSTab>
</OSTabs>
```

### 2. Package Manager Tabs

Shows different package manager commands with automatic preference detection using `CodeTabs`.

<CodeTabs defaultTab="pnpm" group="package-manager">
  <CodeTab label="npm">
    ```bash
    # Using npm
    npm install @leadcms/sdk
    npm run dev
    ```
  </CodeTab>
  <CodeTab label="pnpm">
    ```bash
    # Using pnpm (recommended)
    pnpm add @leadcms/sdk
    pnpm run dev
    ```
  </CodeTab>
  <CodeTab label="yarn">
    ```bash
    # Using yarn
    yarn add @leadcms/sdk
    yarn dev
    ```
  </CodeTab>
  <CodeTab label="bun">
    ```bash
    # Using bun
    bun add @leadcms/sdk
    bun run dev
    ```
  </CodeTab>
</CodeTabs>

**Usage:**
```mdx
<CodeTabs defaultTab="pnpm" group="package-manager">
  <CodeTab label="npm">
    npm install package-name
  </CodeTab>
  <CodeTab label="pnpm">
    pnpm add package-name
  </CodeTab>
  <CodeTab label="yarn">
    yarn add package-name
  </CodeTab>
</CodeTabs>
```

### 4. Custom Tabs (`CodeTabs`)

The base tab component for any tabbed content. All other tab examples above are just `CodeTabs` with different `group` and `defaultTab` values.

<CodeTabs defaultTab="JavaScript" group="language-examples">
  <CodeTab label="JavaScript">
    ```javascript
    // JavaScript example
    import { getCMSContentBySlugForLocale } from '@leadcms/sdk'

    const content = getCMSContentBySlugForLocale('about', 'en')
    console.log(content.metadata.title)
    ```
  </CodeTab>
  <CodeTab label="TypeScript">
    ```typescript
    // TypeScript example
    import { getCMSContentBySlugForLocale, CMSContent } from '@leadcms/sdk'

    const content: CMSContent = getCMSContentBySlugForLocale('about', 'en')
    console.log(content.metadata.title)
    ```
  </CodeTab>
  <CodeTab label="Python">
    ```python
    # Python example (using requests)
    import requests

    response = requests.get('https://your-cms.com/api/content/about')
    content = response.json()
    print(content['title'])
    ```
  </CodeTab>
</CodeTabs>

**Usage:**
```mdx
<CodeTabs defaultTab="Option1" group="my-group">
  <CodeTab label="Option1">
    First option content
  </CodeTab>
  <CodeTab label="Option2">
    Second option content
  </CodeTab>
</CodeTabs>
```

## Advanced Features

### Tab Synchronization

All tab components support grouping, which synchronizes selections across multiple tab instances on the same page.

**Example:** Selecting "pnpm" in one `CodeTabs` with `group="package-manager"` will automatically select "pnpm" in all other tabs with the same group.

**OS Tab Synchronization Demo:**

First installation step - notice your OS is auto-detected and appears first:

<OSTabs>
  <OSTab os="Linux">
    ```bash
    # Step 1: Update package manager (Linux)
    sudo apt update
    sudo apt upgrade
    ```
  </OSTab>
  <OSTab os="macOS">
    ```bash
    # Step 1: Update Homebrew (macOS)
    brew update
    brew upgrade
    ```
  </OSTab>
  <OSTab os="Windows">
    ```powershell
    # Step 1: Update Windows Package Manager
    winget upgrade --all
    ```
  </OSTab>
</OSTabs>

Second installation step - selecting a different OS above will sync this tab too:

<OSTabs>
  <OSTab os="Linux">
    ```bash
    # Step 2: Install Node.js (Linux)
    sudo apt install nodejs npm
    node --version
    ```
  </OSTab>
  <OSTab os="macOS">
    ```bash
    # Step 2: Install Node.js (macOS)
    brew install node
    node --version
    ```
  </OSTab>
  <OSTab os="Windows">
    ```powershell
    # Step 2: Install Node.js (Windows)
    winget install OpenJS.NodeJS
    node --version
    ```
  </OSTab>
</OSTabs>

Third installation step - all three tab groups stay synchronized:

<OSTabs>
  <OSTab os="Linux">
    ```bash
    # Step 3: Install LeadCMS CLI (Linux)
    npm install -g @leadcms/cli
    leadcms --version
    ```
  </OSTab>
  <OSTab os="macOS">
    ```bash
    # Step 3: Install LeadCMS CLI (macOS)
    npm install -g @leadcms/cli
    leadcms --version
    ```
  </OSTab>
  <OSTab os="Windows">
    ```powershell
    # Step 3: Install LeadCMS CLI (Windows)
    npm install -g @leadcms/cli
    leadcms --version
    ```
  </OSTab>
</OSTabs>

**Key Behavior:**
- ✅ **Auto-detection**: Your OS appears first in all tab groups
- ✅ **Synchronization**: Clicking any OS tab updates ALL OS tabs on the page
- ✅ **Persistence**: Your manual selection is remembered across page reloads
- ✅ **Smart ordering**: Even after manual selection, auto-detected OS stays first

### Copy Functionality

All code blocks within tabs automatically include a copy button that appears on hover. This works seamlessly with the global copy functionality.

### Persistence

User selections are automatically saved to localStorage and restored on subsequent visits:

- **OS tabs**: Stored as `code-tabs-os`
- **All CodeTabs**: Stored as `code-tabs-{group}` where `{group}` is the group prop
  - Package manager tabs: `code-tabs-package-manager`
  - Framework tabs: `code-tabs-framework`
  - Language examples: `code-tabs-language-examples`

### Server-Side Rendering

All tab components are designed for SSR compatibility:

- ✅ **Consistent HTML** - Server renders predictable default state
- ✅ **Progressive enhancement** - Client-side logic enhances the experience
- ✅ **No layout shift** - Smooth transition from server to client state

## Best Practices

### 1. Choose the Right Component

- **OSTabs** - For OS-specific commands with automatic detection
- **CodeTabs with group="package-manager"** - For package installation commands
- **CodeTabs with group="framework"** - For framework-specific setup
- **CodeTabs** - For any other tabbed content

### 2. Content Organization

**Good example - Clear, descriptive labels:**
```mdx
<OSTabs>
  <OSTab os="Linux">
    Ubuntu/Debian instructions here
  </OSTab>
  <OSTab os="macOS">
    macOS instructions here
  </OSTab>
  <OSTab os="Windows">
    Windows instructions here
  </OSTab>
</OSTabs>
```

**Avoid - Vague or inconsistent labels:**
```mdx
<CodeTabs>
  <CodeTab label="Option A">
    Content here
  </CodeTab>
  <CodeTab label="Choice B">
    Content here
  </CodeTab>
</CodeTabs>
```

### 3. Consistent Grouping

Use the same group names across related tab instances:

```mdx
<!-- All package manager tabs on this page will sync -->
<CodeTabs defaultTab="pnpm" group="package-manager">
  <CodeTab label="npm">
    npm install package
  </CodeTab>
  <CodeTab label="pnpm">
    pnpm add package
  </CodeTab>
</CodeTabs>

<!-- Later on the same page -->
<CodeTabs defaultTab="pnpm" group="package-manager">
  <CodeTab label="npm">
    npm run build
  </CodeTab>
  <CodeTab label="pnpm">
    pnpm run build
  </CodeTab>
</CodeTabs>
```

### 4. Accessibility

All tab components include proper ARIA attributes and keyboard navigation support. They follow WAI-ARIA tab panel patterns for screen reader compatibility.
