# Installation

This guide covers different ways to install and set up LeadCMS depending on your needs and environment.

## Quick Start (Recommended)

The fastest way to get started with LeadCMS:

```bash
npx @leadcms/create-app my-app
cd my-app
npm run dev
```

This creates a new LeadCMS project with:
- Next.js frontend
- TypeScript configuration
- Tailwind CSS for styling
- Example content types
- Development server setup

## Framework Integration

LeadCMS works with any frontend framework. Here are setup guides for popular options:

### Next.js

<Steps>
  <Step title="Install Dependencies">
    ```bash
    npm install @leadcms/client @leadcms/nextjs-adapter
    ```
  </Step>
  <Step title="Configure Next.js">
    Create or update `next.config.js`:
    ```javascript
    const { withLeadCMS } = require('@leadcms/nextjs-adapter')

    module.exports = withLeadCMS({
      // Your Next.js config
      reactStrictMode: true,
      images: {
        domains: ['cdn.leadcms.io']
      }
    })
    ```
  </Step>
  <Step title="Add Environment Variables">
    Create `.env.local`:
    ```bash
    LEADCMS_API_KEY=your_api_key
    LEADCMS_PROJECT_ID=your_project_id
    LEADCMS_ENVIRONMENT=development
    ```
  </Step>
</Steps>

### React (Vite)

<Steps>
  <Step title="Install Dependencies">
    ```bash
    npm install @leadcms/client @leadcms/react
    ```
  </Step>
  <Step title="Configure Vite">
    Update `vite.config.ts`:
    ```typescript
    import { defineConfig } from 'vite'
    import react from '@vitejs/plugin-react'
    import { leadcms } from '@leadcms/vite-plugin'

    export default defineConfig({
      plugins: [
        react(),
        leadcms({
          apiKey: process.env.VITE_LEADCMS_API_KEY,
          projectId: process.env.VITE_LEADCMS_PROJECT_ID
        })
      ]
    })
    ```
  </Step>
</Steps>

### Vue.js

<Steps>
  <Step title="Install Dependencies">
    ```bash
    npm install @leadcms/client @leadcms/vue
    ```
  </Step>
  <Step title="Configure Vue">
    In your `main.js`:
    ```javascript
    import { createApp } from 'vue'
    import { createLeadCMS } from '@leadcms/vue'
    import App from './App.vue'

    const leadcms = createLeadCMS({
      apiKey: process.env.VUE_APP_LEADCMS_API_KEY,
      projectId: process.env.VUE_APP_LEADCMS_PROJECT_ID
    })

    createApp(App)
      .use(leadcms)
      .mount('#app')
    ```
  </Step>
</Steps>

## Environment Setup

### Development Environment

For development, set up these environment variables:

```bash
# .env.local (for Next.js) or .env (for other frameworks)
LEADCMS_API_KEY=your_development_api_key
LEADCMS_PROJECT_ID=your_project_id
LEADCMS_ENVIRONMENT=development
LEADCMS_PREVIEW_MODE=true
LEADCMS_API_URL=https://api.leadcms.io/v1
```

### Production Environment

For production deployment:

```bash
# Production environment variables
LEADCMS_API_KEY=your_production_api_key
LEADCMS_PROJECT_ID=your_project_id
LEADCMS_ENVIRONMENT=production
LEADCMS_PREVIEW_MODE=false
LEADCMS_CDN_URL=https://cdn.leadcms.io
```

<Callout type="warning" title="API Key Security">
Use different API keys for development and production. Production keys should have limited permissions and be stored securely.
</Callout>

## Docker Installation

Run LeadCMS in a Docker container:

<Steps>
  <Step title="Create Dockerfile">
    ```dockerfile
    FROM node:18-alpine

    WORKDIR /app

    COPY package*.json ./
    RUN npm ci --only=production

    COPY . .
    RUN npm run build

    EXPOSE 3000
    CMD ["npm", "start"]
    ```
  </Step>
  <Step title="Create docker-compose.yml">
    ```yaml
    version: '3.8'
    services:
      app:
        build: .
        ports:
          - "3000:3000"
        environment:
          - LEADCMS_API_KEY=${LEADCMS_API_KEY}
          - LEADCMS_PROJECT_ID=${LEADCMS_PROJECT_ID}
        volumes:
          - ./.env.local:/app/.env.local:ro
    ```
  </Step>
  <Step title="Build and Run">
    ```bash
    docker-compose up --build
    ```
  </Step>
</Steps>

## Self-Hosted Setup

For self-hosted installations, you'll need to set up the LeadCMS backend:

<Callout type="info" title="Self-Hosting Requirements">
Self-hosting requires:
- Node.js 18+
- PostgreSQL 13+
- Redis 6+
- S3-compatible storage (AWS S3, MinIO, etc.)
</Callout>

<Steps>
  <Step title="Clone Repository">
    ```bash
    git clone https://github.com/leadcms/leadcms-server.git
    cd leadcms-server
    ```
  </Step>
  <Step title="Install Dependencies">
    ```bash
    npm install
    ```
  </Step>
  <Step title="Configure Environment">
    ```bash
    cp .env.example .env
    # Edit .env with your database and storage settings
    ```
  </Step>
  <Step title="Run Migrations">
    ```bash
    npm run migrate
    ```
  </Step>
  <Step title="Start Server">
    ```bash
    npm run start:prod
    ```
  </Step>
</Steps>

## Verification

After installation, verify everything is working:

### Check Installation

```bash
# Check CLI version
leadcms --version

# Check if project is properly configured
leadcms info
```

### Test API Connection

```javascript
// test-connection.js
const { LeadCMS } = require('@leadcms/client')

const cms = new LeadCMS({
  apiKey: process.env.LEADCMS_API_KEY,
  projectId: process.env.LEADCMS_PROJECT_ID
})

cms.projects.current()
  .then(project => {
    console.log('✅ Connected to LeadCMS:', project.name)
  })
  .catch(error => {
    console.error('❌ Connection failed:', error.message)
  })
```

Run the test:

```bash
node test-connection.js
```

## Troubleshooting

### Common Issues

**Module not found errors:**
```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```

**API connection issues:**
- Verify your API key and project ID
- Check if your IP is whitelisted (for production)
- Ensure environment variables are properly set

**Build errors:**
- Check Node.js version compatibility
- Verify all dependencies are installed
- Review TypeScript configuration if using TypeScript

### Getting Help

If you encounter issues:

1. Check our [troubleshooting guide](/docs/troubleshooting)
2. Search [GitHub issues](https://github.com/leadcms/leadcms/issues)
3. Join our [Discord community](https://discord.gg/leadcms)
