Most businesses use separate tools for managing content and customers: a CMS for the website and a CRM for leads and customers. This creates data silos, integration headaches, and unnecessary complexity. What if your CMS could do both?

## The Problem with Separate Systems

### Data Silos

```
Traditional Setup:
┌──────────────┐         ┌──────────────┐
│   CMS        │         │   CRM        │
│   (Website)  │   ???   │   (Leads)    │
├──────────────┤         ├──────────────┤
│ • Content    │         │ • Contacts   │
│ • Pages      │         │ • Deals      │
│ • Forms      │         │ • Pipelines  │
└──────────────┘         └──────────────┘

Problem: Data doesn't flow between systems
```

When someone fills out a form on your website:
1. Form submission goes to CMS
2. Must be manually exported
3. Imported into CRM
4. Or via expensive integration tools

### Integration Complexity

**Zapier/Make Workarounds**:
```typescript
// Fragile integration chain
Website Form → Webhook → Zapier →
Transform Data → CRM API → Hope It Works
```

Issues:
- Monthly costs ($20-100/month)
- Breaking changes when APIs update
- Data format mismatches
- Delayed syncing
- Debugging nightmares

### Double the Tools, Double the Cost

```
Monthly Costs (typical):
CMS (Contentful/Sanity): $99-299/month
CRM (HubSpot/Salesforce): $50-200/month
Integration (Zapier): $20-50/month
───────────────────────────────────────
Total: $169-549/month ($2,028-6,588/year)
```

When a visitor submits a form, everything happens automatically:


Behind the scenes:
1. Form submitted on website
2. Lead created in CRM instantly

All in one system.

## Real-World Use Cases

### SaaS Company

```typescript
// Lead capture from multiple sources
// Free trial signup
app.post('/api/trial', async (req, res) => {
  const { email, name, plan } = req.body

  // Creates both user account AND CRM lead
  const user = await createUser({ email, name })
  const lead = await createLead({
    email,
    name,
    source: 'Trial Signup',
    plan,
    status: 'Trial',
    score: 25
  })

  // Trigger automation
  await sendTrialWelcomeEmail(user)
  await assignToSalesRep(lead)
})

// Track product usage
app.post('/api/track', async (req, res) => {
  const { userId, event } = req.body

  // Update lead score based on usage
  if (event === 'feature_used') {
    await incrementLeadScore(userId, 5)
  }
  if (event === 'invited_team_member') {
    await incrementLeadScore(userId, 15)
  }
})
```

### E-commerce Store

```typescript
// Abandoned cart recovery
async function trackCartAbandonment() {
  const carts = await getAbandonedCarts()

  for (const cart of carts) {
    // Create/update lead
    await updateLead(cart.email, {
      status: 'Abandoned Cart',
      cartValue: cart.total,

      abandonedAt: cart.lastActivity
```

## Getting Started

### Enable CRM Features

```typescript
// leadcms.config.ts
export default {
  crm: {
    enabled: true,
    features: {
      contacts: true,
      leads: true,
      deals: true,
      pipeline: true,
      emailTracking: true,
      automation: true
    }
  }
}
```

### Create First Form

```typescript
// Contact form with CRM integration
import { ContactForm } from '@leadcms/components'

<ContactForm
  fields={[
    { name: 'name', required: true },
```
